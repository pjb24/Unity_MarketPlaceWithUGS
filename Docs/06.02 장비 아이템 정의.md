## 1. 최상위 메타
### schema
- **JSON 구조 버전**
- 파싱/마이그레이션 기준점
- 서버는 schema로 분기 처리
---
### kind
- 아이템 종류 구분자
- `"EQ"` 고정
- FRAG/EQ 공통 파서에서 분기용
---
### slot
- 장비 타입
- `"WPN" | "ARM" | "ACC"`
- 착용 슬롯 결정
- 제작 시 **Core 슬롯에 넣은 파편의 slot 값**
---
### tier
- 장비 티어
- 제작/분해/강화/각성 비용 기준
- 거래소 가격대 분기 기준
---
### rarity
- 레어리티
- 드랍 확률 / 분해 수량 / 시장 희소성용
- **스탯 계산에는 직접 영향 없음**
---
### state
- 장비의 현재 상태(enum)
- `"NORMAL" | "AWAKENED" | "EXPIRED"`
- 시즌 기록 가능 여부, 변경 가능 여부 판단의 핵심 키
---
## 2. 생명주기 정보
### lifecycle
#### createdAt
- 장비 인스턴스 생성 시각
- 로그/감사/CS용
#### createdBy
- 생성 경로
- `"CRAFT" | "DROP" | "REWARD" | "ADMIN"`
- 부정 생성 추적용
#### updatedAt
- 마지막 상태 변경 시각
- 각성/강화/시즌 전이 시 갱신
---
## 3. 시즌 관련 정보
### season
#### craftedSeason
- 장비가 제작된 시즌
#### awakenedSeason
- 각성 완료 시즌
- null이면 미각성
- 시즌 제한 규칙의 기준점
#### expireAfterSeasons
- 각성 후 **몇 시즌까지 기록 허용**
- 예: 2 → 각성 시즌 + 2 시즌까지
#### expiredAtSeason
- EXPIRED로 전이된 시즌
- 상태 전이 로그/검증용
#### recordEligibleState
- 기록 등록이 허용되는 상태
- 보통 "AWAKENED"
#### recordEligibleUntilSeason
- 기록 허용 마지막 시즌
- **계산 결과를 저장한 값**
- 실시간 계산을 피하기 위한 캐시
---
## 4. 시즌 효과 정보
### seasonEffect
- 장비가 만들어질 때 어떤 시즌 규칙이 적용됐고, 그 결과 어떤 보정이 실제로 들어갔는지를 기록한 고정 스냅샷
#### appliedSeason
- **장비 제작이 이루어진 시즌**
- 시즌 효과 판단의 기준점
#### ruleSetId
- 적용된 **시즌 규칙 세트 ID**
- 어떤 시즌 룰이 사용됐는지 식별용
#### applied
- 시즌 효과 적용 여부
- 조건 불충족 / 추가 규칙 없음이면 `false`
#### modifiers
- **실제로 적용된 수치 보정 결과**
- 제작 스탯 계산에 이미 반영된 값
- 설명·검증용 스냅샷
#### hash
- 시즌 효과 데이터의 무결성 해시
- 변조/오염 검증용
## 5. 시즌 기록 판단 결과
### record
#### eligible
- 현재 시즌 기준 **기록 등록 가능 여부**
- UI 즉시 판단용
#### eligibleReason
- 불가 사유 코드
- 예:
    - `NOT_AWAKENED`, `SKILL_NOT_SELECTED`, `EXPIRED`, `LOCKED`, `AWAKENED`
- CS/로그 분석에 필수
---
## 6. 강화 정보
### enhance
#### level
- 현재 강화 단계
#### maxLevelBeforeAwaken
- 각성 가능 최대 강화 단계
- 각성 조건 체크용
---
## 7. 스탯 정보 (핵심)
### stats
#### base
- **제작 직후 강화 0 기준 스탯**
- 절대 변경 금지
- 모든 재계산의 기준
#### now
- 현재 적용 스탯
- 강화/상태 반영 결과
- 파생값(재계산 가능)
#### awakenedFinal
- 각성 완료 시 최종 확정 스탯
- 각성 전 UI 사전 공개용
- 절대 변경 금지
---
## 8. 스킬 정보
### skill
#### candidates
- 제작에 사용된 파편 3개의 스킬 목록
- 변경 불가
- 제작 결과의 근거 데이터
#### chosen
- 최종 선택된 스킬 ID
- 스킬 선택 시 1회 확정
#### chosenFromFragmentInstanceId
- 어떤 파편에서 선택된 스킬인지
- 감사/재현/NFT용
#### status
- 스킬 선택 진행 상태.
- 값(권장 고정):
    - `PENDING` : 후보는 있으나 아직 확정 선택 전
    - `SELECTED` : 확정 선택 완료(이후 변경 불가)
#### selectedAt
- 스킬을 **확정한 시각(ISO8601)**.
- `PENDING`이면 `null`.
- CS/감사/버그 추적(“언제 선택됐나”)에 필요.
#### locked
- 스킬 선택 데이터의 **불변성 플래그**.
- `SELECTED`가 되면 `true`.
- 서버는 `locked == true`면 `chosen`, `chosenFromFragmentInstanceId`, `status`, `selectedAt` 변경을 거부해야 한다.
---
## 9. 제작 정보
### craft
#### craftedAt
- 제작 시각
#### sourceFragmentInstanceIds
- 사용된 파편 인스턴스 ID 3개
- **장비 정체성의 핵심**
- 절대 변경 금지
#### slots.core
- Core 슬롯에 배치된 파편 ID
#### slots.free
- Free 슬롯에 배치된 파편 ID 2개
---
## 10. 각성 정보
### awaken
#### effectId
- 각성 특수 효과 ID
#### effectHash
- 각성 효과 해시
- NFT/무결성 검증용
#### lockedAfterAwaken
- 각성 후 변경 불가 플래그
- 안전장치
---
## 11. EXPIRED 전용 정보
### expired
#### extraEffectId
- EXPIRED 상태에서 추가로 적용되는 특수 효과
#### extraEffectHash
- 해당 효과의 해시
#### mode
- `"ADD"`: 각성 효과 + 추가 효과
- `"REPLACE"`: 각성 효과 대체
#### appliesOnlyWhenStateIsExpired
- 안전 플래그
- 상태 불일치 시 효과 적용 방지
---
## 12. 거래소 관련
### market
#### tradable
- 거래 가능 여부
#### tradeLock.isLocked
- 강제 거래 잠금
#### tradeLock.reason
- 잠금 사유 (환불/제재/오류)
- `reason`은 **사람이 읽는 문장 말고 코드**로 저장해라. (로그/분석/대응 쉬움)
##### 운영/CS
- `CS_HOLD` : CS 처리 중 임시 보류
- `CHARGEBACK_REVIEW` : 결제 차지백/분쟁 연관 조사
- `FRAUD_REVIEW` : 부정행위 조사 보류
- `POLICY_VIOLATION` : 정책 위반 제재
##### 데이터/시스템
- `DATA_MIGRATION` : 마이그레이션 중 잠금
- `DUPLICATE_DETECTED` : 중복 인스턴스 의심
- `INTEGRITY_CHECK_FAIL` : 무결성 검증 실패(해시 불일치 등)
- `ROLLBACK_PENDING` : 거래/정산 롤백 대기
- `REFUND_PENDING` : 환불 처리 대기
- `ESCROW_INCONSISTENT` : 에스크로 상태 불일치
##### 게임 규칙
- `EQUIPPED_LOCK` : 착용 상태로 거래 불가
- `SEASON_RECORD_LOCK` : 시즌 기록 등록 상태라 거래 제한
- `AWAKEN_TRANSITION` : 각성 처리 중 임시 잠금
- `EXPIRED_TRANSITION` : EXPIRED 전이 처리 중 임시 잠금
> 같이 쓰면 좋은 패턴
> - `reason`: 코드 1개
> - `location.note` 또는 별도 `tradeLock.note`: 사람이 읽는 상세 메모
#### tradeLock.until
- 잠금 해제 시각 (null이면 수동 해제)
---
## 13. 위치 정보
### location
#### zone
- 아이템 현재 위치
- `"BAG" | "EQUIPPED" | "MARKET_ESCROW" | "STORAGE"`
#### note
- 디버그/운영 메모
- `location.zone`만으로 부족한 **상세 사유/상태**를 텍스트로 남기는 용도다.  
(운영/디버그/복구에 쓰고, 로직 분기는 여기 의존하지 마라)
예시:
- `"Equipped by preset: PvE_Farm_01"`
- `"Moved to MARKET_ESCROW for listing L10001"`
- `"Recovered after disconnect during buy flow (txn:TX123)"`
- `"Auto-moved to STORAGE due to inventory overflow"`
- `"Temporary parking for migration batch #42"`
- `"Returned to BAG after listing expired (L10001)"`
- `"Locked item: investigation case 2026-01-23-771"`

- NFT 발행 가능 여부

## 16. 기타
### flags
- 임시/실험/운영 플래그용
- 로직에 직접 의존하지 말 것
#### `flags` 예시
`flags`는 **임시/운영 보조 플래그**를 담는다.  
원칙: “있으면 편하지만 없어도 시스템이 동작”해야 한다.
권장 예시:
##### UX/튜토리얼/알림
- `isNew`: true (신규 획득 표시)
- `highlightInInventory`: true (UI 강조)
- `showSkillSelectHint`: true (스킬 선택 튜토리얼 띄우기)
##### 거래소/경제 보조
- `marketExpiredBonusEligible`: true (만료 후 분해 보너스 대상)
- `marketFeeDiscountApplied`: true (특정 이벤트 수수료 적용 여부)
- `priceFloorProtected`: true (운영이 하한가 보호 처리)
##### 안정성/복구
- `needsRecalcStatsNow`: true (`stats.now` 재계산 필요)
- `needsProofRehash`: true (`proof.snapshotHash` 재생성 필요)
- `quarantined`: true (격리 상태, UI 노출/거래 제한과 연동 가능)
##### 이벤트/시즌 운영
- `eventTagged`: "EVT_2026_LUNAR" (이벤트 태그)
- `seasonLegacyBonusApplied`: true (시즌 종료 보너스 적용 여부)
> 주의  
> `flags`는 시간이 지나면 쓰레기장이 된다.  
> **키 목록을 코드에서 상수로 고정**하고, 안 쓰는 플래그는 제거하라.
