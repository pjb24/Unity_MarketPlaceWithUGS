# 장비 파편 아이템 ID
- `FRAG_{SLOT}_{TIER}`
- 예: `FRAG_WPN_T1`, `FRAG_ARM_T2`, `FRAG_ACC_T3`
```json
{
  "schema": 1,

  "kind": "FRAG",

  "slot": "WPN",
  "tier": 1,
  "rarity": "COMMON",

  "skillId": "",

  "stats": {
    "base": {},
    "potentialCaps": {}
  },

  "lifecycle": {
    "createdAt": "",
    "createdBy": "DROP",
    "updatedAt": ""
  },

  "market": {
    "tradable": true,
    "tradeLock": {
      "isLocked": false,
      "reason": null,
      "until": null
    }
  },

  "location": {
    "zone": "BAG",
    "note": null
  },

  "proof": {
    "snapshotHash": null,
    "buildHash": null
  },

  "flags": {
		"marketExpiredBonusEligible": false
  }
}
```
## 1. 최상위 메타
### schema
- JSON 구조 버전.
- 파싱/마이그레이션 기준.
- 구조 변경 시 반드시 증가.
---
### kind
- 아이템 종류 식별자.
- `"FRAG"` 고정.
- EQ/FRAG 공용 파서 분기용.
---
## 2. 파편 기본 분류 정보
### slot
- 파편 타입.
- `"WPN" | "ARM" | "ACC"`.
- **제작 시 Core 슬롯에 들어가면 장비 타입을 결정**.
---
### tier
- 파편 티어.
- 드랍 테이블, 분해 재료량, 재제조 비용, 거래소 가격대 기준.
- 파편 가치의 1차 기준축.
---
### rarity
- 파편 희귀도.
- 드랍 확률, 분해 수량 보정, 시장 희소성 표현용.
- **스탯 계산에는 직접 영향 없음**.
---
## 3. 스킬 / 스탯 핵심 데이터
### skillId
- 파편이 제공하는 **고유 스킬 1종**.
- 장비 제작 시 **스킬 후보(candidate)** 로만 사용됨.
- 생성 후 변경 금지.
---
### stats
#### base
- 파편의 **기본 스탯 기여분**.
- 장비 스탯 계산 시 합산 대상.
- 생성 후 절대 변경 금지.
#### potentialCaps
- 각성 시 적용될 **잠재 최대치 계산 근거**.
- “각성 후 최종 스탯이 왜 이 값인지”의 근거 데이터.
- 생성 후 절대 변경 금지.
---
## 4. 생명주기 관리
### lifecycle
#### createdAt
- 파편 생성 시각.
- 로그/CS/감사용.
#### createdBy
- 생성 경로.
- `"DROP" | "CRAFT" | "REWARD" | "ADMIN"`.
- 비정상 생성 추적용.
#### updatedAt
- 마지막 상태 변경 시각.
- 거래소 이동, 잠금 처리, 복구 시 갱신.
---
## 5. 거래소/운영 제어
### market
- 파편의 거래 가능 상태 관리.
#### tradable
- 거래 가능 여부.
- 이벤트/정책으로 일시 제한 가능.
#### tradeLock
- 강제 거래 잠금 정보.
#### tradeLock.isLocked
- true면 거래 불가.
#### tradeLock.reason
- 잠금 사유 코드(CS, 조사, 복구 등).
#### tradeLock.until
- 잠금 해제 시각(null이면 수동 해제).
> 거래소 에스크로/정산 문제 발생 시 **파편 단위 잠금**이 필요함.
---
## 6. 위치 정보
### location
#### zone
- 아이템 현재 위치
- `"BAG" | "EQUIPPED" | "MARKET_ESCROW" | "STORAGE"`
#### note
- 디버그/운영 메모
- `location.zone`만으로 부족한 **상세 사유/상태**를 텍스트로 남기는 용도다.  
    (운영/디버그/복구에 쓰고, 로직 분기는 여기 의존하지 마라)  
    예시:
- `"Equipped by preset: PvE_Farm_01"`
- `"Moved to MARKET_ESCROW for listing L10001"`
- `"Recovered after disconnect during buy flow (txn:TX123)"`
- `"Auto-moved to STORAGE due to inventory overflow"`
- `"Temporary parking for migration batch #42"`
- `"Returned to BAG after listing expired (L10001)"`
- `"Locked item: investigation case 2026-01-23-771"`
---
## 7. 무결성/검증
### proof
- 데이터 변조/중복/오염 추적용.
#### snapshotHash
- 핵심 필드 스냅샷 해시.
- “이 파편이 왜 이런 스탯/스킬인지” 증명용.
#### buildHash
- 생성 당시 게임 빌드 해시.
- 패치/밸런스 변경 시점 추적.
---
## 8. 플래그 영역
### flags
임시/보조 상태 플래그 모음.
> 로직의 핵심 분기는 여기 의존하지 말 것.
#### marketExpiredBonusEligible
- 거래소 미판매 만료 후 분해 시 **보너스 적용 대상 여부**.
